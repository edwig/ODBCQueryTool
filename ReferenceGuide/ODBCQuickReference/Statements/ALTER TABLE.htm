ALTER TABLE

Use this command to alter the definition of a table in one of the following ways:

  • to add one or more a column(s)
  • to drop a column
  • to alter the definition of a column
  • to add an integrity constraint to the table
  • to alter the default value for a column
  • to enable, disable, or drop an integrity constraint
 

Syntax of the SQL statement

ALTER TABLE base-table-name

{

   ADD   column data-type

| ADD (column data-type [, ...])

| ADD   column data-type DEFAULT literal-value

| ADD   column data-type column-constraint

| ADD   column data-type COLLATION collation

| ADD   column data-type [DEFAULT literal] [column-constraint] [COLLATION collation]

| ALTER [COLUMN] column SET   DEFAULT literal-value

| ALTER [COLUMN] column DROP DEFAULT

| DROP   [COLUMN] column [CASCADE | RESTRICT ]

| DROP CONSTRAINT constraint-name [CASCADE | RESTRICT ]

| ADD   table-constraint

}

  table-constraint ::=

 

[ CONSTRAINT constraint-name]

{

   UNIQUE (column [, ...] )

| PRIMARY KEY (column [, ...])

| FOREIGN KEY (column [, ...]) REFERENCES referenced-table-name (column [, ...])

| CHECK (search-condition)

}

[ constraint-attributes ]

column-constraint ::=

 

[ CONSTRAINT constraint-name]

{

   PRIMARY KEY (column [, ...])

| FOREIGN KEY (column [, ...]) REFERENCES referenced-table-name (column [, ...])

}

[ constraint-attributes ]

constraint-attributes ::=

 

   constraint-check-time [[NOT] DEFERRABLE]

| [NOT] DEFERRABLE [constraint-check-time]

 

constraint-check-time ::=

   INITIALLY DEFERRED

| INITIALLY IMMEDIATE
 

To see if a particular syntax is supported consult the ODBC tree for the following items:

- The SQL conformance level

- The ODBC conformance level

- 'Alter table options' under 'SQL support'. This info is rather unreliable for some database vendors.
